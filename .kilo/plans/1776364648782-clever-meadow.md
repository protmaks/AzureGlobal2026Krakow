## Module Reference Information

### Module Git Paths (from pchylak/global_azure_2026_ccoe)
All modules use: `git::https://github.com/pchylak/global_azure_2026_ccoe.git?ref=<module-name>/v1.0.0`

### Module Configuration Details

#### Managed Identity Module
- **Path**: managed_identity/v1.0.0
- **Required variables**: name, resource_group (with location and name)
- **Optional variables**: permissions list (for role assignments), tags
- **Output**: principal_id (needed for Key Vault & SQL access), client_id

#### SQL Server Module  
- **Path**: mssql_server/v1.0.0
- **Required**: sql_server_name, resource_group, sql_server_admin, sql_server_version
- **Optional**: databases list (name, size, sku, storage_account_type, collation), public_network_access (default: true)
- **Note**: Will need to add firewall rule to allow App Service access or use service endpoint

#### Application Insights Module
- **Path**: application_insights/v1.0.0
- **Required**: log_analytics_name, application_insights_name, resource_group
- **Optional**: retention_in_days (default: 30), tags
- **Output**: instrumentation_key, connection_string

#### App Service Plan Module
- **Path**: service_plan/v1.0.0
- **Required**: app_service_plan_name, sku_name (for B1: "B1"), resource_group
- **Optional**: tags
- **Output**: id (needed for App Service module)

#### App Service Module
- **Path**: app_service/v1.0.0
- **Required**: app_service_name, app_service_plan_id, identity_id, identity_client_id, app_settings, resource_group
- **Optional**: health_check_path, health_check_eviction_time_in_min, ftps, always_on, https_only, http2_enabled, tags
- **App Settings**: Must include ConnectionStrings__RazorPagesMovieContext and ApplicationInsights__ConnectionString

#### Container Registry Module
- **Path**: container_registry/v1.0.0
- **Required**: container_registry_name, resource_group
- **Optional**: sku (default: "Basic"), read_access (list of principal IDs), write_access (list of principal IDs), tags
- **Note**: Managed Identity needs AcrPull role for ACR access

#### Key Vault Module (Already exists)
- **Path**: keyvault/v1.0.0
- **Currently configured**: keyvault_name, resource_group, network_acls
- **To add**: secrets list (for DB connection string & App Insights key), permissions for Managed Identity

## App Service Docker Configuration
The App Service needs to be configured to use Docker from ACR:
- Docker image URL pattern: `{acr_name}.azurecr.io/{image_name}:{tag}`
- Registry authentication: Managed Identity (preferred) or username/password
- Environment variables must reference Key Vault using `@Microsoft.KeyVault(SecretUri=...)`

## Key Vault Secret Format
For App Service to access Key Vault secrets, use:
```
"ConnectionStrings__RazorPagesMovieContext" = "@Microsoft.KeyVault(SecretUri=https://{keyvault_name}.vault.azure.net/secrets/{secret_name}/)"
```

## Pipeline Workflow Jobs
1. **build-and-push**: Docker build & push to ACR (needs Azure login with ACR push permissions)
2. **deploy-infra**: Terraform init/plan/apply (needs Azure authentication for resource creation)
3. **update-app-service**: Deploy container to App Service (uses AzureWebAppContainer@1 action)

## Notes
- The terraform.yml workflow uses basic auth (client secret) - consider migrating to OIDC like deploy.yml
- deploy.yml already uses OIDC (ARM_USE_OIDC=true), which is more secure
- All pipeline jobs should use consistent authentication method
- Database migrations run on app startup (Program.cs checks connection string and runs context.Database.Migrate())
- App needs network connectivity to both SQL Server and Key Vault (both should allow Azure Services in their firewall rules)
